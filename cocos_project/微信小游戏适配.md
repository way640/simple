# 微信小游戏适配指南

## 概述
微信小游戏平台对游戏性能、包大小和用户体验有特殊要求。为确保《幸存者：卡牌进化》能在微信小游戏中顺利运行，需要遵循以下适配指南。

## 性能要求

### 帧率要求
- **目标帧率**: 不低于30FPS
- **最低帧率**: 不低于20FPS

### 内存限制
- **总内存**: 不超过200MB
- **图片内存**: 不超过100MB
- **音频内存**: 不超过50MB

### 加载时间
- **首次加载**: 不超过5秒
- **场景切换**: 不超过2秒
- **资源加载**: 不超过3秒

## 包大小限制

### 图片文件
- **最大尺寸**: 不超过100KB
- **建议尺寸**: 不超过50KB
- **图片格式**: PNG（带透明度）或 JPEG（不带透明度）

### 音频文件
- **最大尺寸**: 不超过200KB
- **建议尺寸**: 不超过100KB
- **音频格式**: MP3或WAV

### 游戏总包大小
- **最大尺寸**: 不超过10MB
- **建议尺寸**: 不超过8MB
- **压缩要求**: 必须使用微信小游戏打包工具压缩

## 优化策略

### 图片优化
1. **Sprite Sheet使用**
   - 将所有角色动画合并到Sprite Sheet中
   - 将所有怪物动画合并到Sprite Sheet中
   - 将所有UI图标合并到Sprite Sheet中

2. **图片压缩**
   - 使用pngquant压缩PNG图片
   - 使用jpegoptim压缩JPEG图片
   - 减少图片分辨率（128x128或64x64）

### 音频优化
1. **音频压缩**
   - 使用mp3或wav格式
   - 降低音频采样率（44100Hz降低到22050Hz）
   - 减少音频比特率（320kbps降低到128kbps）

2. **音频分段**
   - 将长音频分段为短音频
   - 只加载必要音频
   - 延迟加载音频资源

### 代码优化
1. **代码压缩**
   - 使用微信小游戏打包工具压缩代码
   - 删除无用代码
   - 减少代码体积

2. **内存优化**
   - 使用单例模式减少内存占用
   - 使用对象池减少对象创建
   - 使用缓存减少重复计算

## 注意事项

### 微信限制
1. **不能使用eval**
   ```
   // 不能使用eval
   eval('console.log("hello")');
   ```

2. **不能使用Function**
   ```
   // 不能使用Function
   new Function('console.log("hello")');
   ```

3. **不能使用global**
   ```
   // 不能使用global
   global['test'] = 'test';
   ```

### 微信要求
1. **必须有分享功能**
   ```
   // 必须有分享功能
   wx.shareAppMessage();
   ```

2. **必须有登录功能**
   ```
   // 必须有登录功能
   wx.login();
   ```

3. **必须有支付功能**
   ```
   // 必须有支付功能
   wx.requestPayment();
   ```
